Overview

This section presents the machine learning models implemented for our project. To satisfy the project requirement, we selected different models for each of our dataset: Frequent Pattern Mining, Classification, Clustering, and Regression. Each model was chosen based on the structure of the dataset, the type of question being answered, and the preprocessing steps required before training or analysis.

Frequent Pattern Mining

Apriori / FP-Growth

Used to identify recurring co-occurrence patterns in transformed categorical or binned observational records.

Classification

Decision Tree / SVM

Used to classify observations such as species category, anomaly class, or environmental grouping based on engineered features.

Clustering

K-Means / DBSCAN

Used to discover natural groupings in the data without labels, especially after feature scaling and dimensionality reduction.

Regression

Linear Regression

Used to model and predict continuous outcomes such as temperature or count-related trends from numerical predictors.

3. Frequent Pattern Mining

Suggested Model

Apriori (or FP-Growth)

  • Why this model was chosen: Frequent pattern mining is useful when the goal is to discover repeated relationships among categorical conditions, such as combinations of species, season, count range, or climate category.
  • Dataset fit: Because our raw data is largely numerical and observational, we first converted selected features into categories or bins so that records could be treated like transactions.
  • Model assumptions: The model assumes that meaningful co-occurrence patterns exist and that data has been transformed into an itemset/transaction structure.
  • Hyperparameter tuning: Minimum support and minimum confidence were adjusted to balance too many trivial rules versus too few rules.
  • Challenges & solutions: Continuous variables were not directly suitable, so we discretized them into interpretable bins such as low/medium/high temperature or count categories.

4. Classification

Suggested Model

Decision Tree Classifier

  • Why this model was chosen: Decision Trees are easy to interpret and work well when the goal is to explain how features separate observations into classes.
  • Dataset fit: This model is helpful when predicting a categorical outcome, such as species group, anomaly type, or class label derived from the data.
  • Model assumptions: Decision Trees do not require linearity and can model nonlinear splits, though they may overfit if depth is not controlled.
  • Hyperparameter tuning: We adjusted parameters such as max_depth, min_samples_split, and min_samples_leaf to improve generalization.
  • Challenges & solutions: Some features needed encoding and class imbalance had to be monitored so one class did not dominate predictions.

5. Clustering

Suggested Model

K-Means Clustering

  • Why this model was chosen: K-Means is useful for identifying hidden structure in unlabeled data and grouping observations with similar feature values.
  • Dataset fit: Our cleaned numerical variables can be grouped to explore whether environmental or biological observations naturally form clusters.
  • Model assumptions: K-Means assumes compact, roughly spherical clusters and is sensitive to scale, so feature standardization is important.
  • Hyperparameter tuning: We tested different values of k and used metrics such as silhouette score and visual inspection to choose the most meaningful cluster count.
  • Challenges & solutions: Variables had different scales, so we applied standardization first. We also checked whether PCA helped improve interpretability in plots.

6. Regression

Suggested Model

Linear Regression

  • Why this model was chosen: Linear Regression is appropriate when predicting a continuous target and establishing a baseline for how well input variables explain change in the outcome.
  • Dataset fit: This model is well suited for numerical targets such as temperature, sea ice extent, or transformed penguin count trends.
  • Model assumptions: Linear Regression assumes a linear relationship, independent errors, constant variance, and limited multicollinearity among predictors.
  • Hyperparameter tuning: Basic Linear Regression has limited tuning, so improvement came mainly from feature engineering such as lag variables, scaling, or removing weak predictors.
  • Challenges & solutions: Since the data has time-based structure, we created lag features and used a non-shuffled train/test split to preserve temporal order.
